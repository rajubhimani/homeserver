Ubuntu · Self-hosting

Expose home services to the internet

Step-by-step guide for Immich, Nextcloud, and any other service running on your home server.

Browser→ Cloudflare DNS→ Encrypted Tunnel→ cloudflared daemon→ Your service
✓ No port forwarding ✓ Works behind CGNAT ✓ Free SSL auto ✓ Recommended for beginners
1
Add your domain to Cloudflare
▼
Go to cloudflare.com → create a free account → click Add a Site → enter your domain → choose the Free plan.
Cloudflare will give you 2 nameservers like:
art.ns.cloudflare.com and bob.ns.cloudflare.com
Go to your domain registrar (GoDaddy, Namecheap, etc.) and replace the existing nameservers with these two. Wait 10–30 minutes for it to propagate.
After this, Cloudflare controls your domain's DNS. All future DNS changes happen in the Cloudflare dashboard.
2
Install cloudflared on your Ubuntu server
▼
SSH into your server and run these commands one by one:
# Download the package
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb

# Install it
sudo dpkg -i cloudflared.deb

# Verify installation
cloudflared --version
3
Login to Cloudflare from your server
▼
Run this command — it will print a URL in the terminal. Copy the URL, open it in your browser, and authorize your domain.
cloudflared tunnel login
This links your server to your Cloudflare account and saves a credentials file.
4
Create a tunnel
▼
Give your tunnel any name — here we use home-tunnel.
cloudflared tunnel create home-tunnel
This prints a Tunnel ID that looks like: 6ff42ae2-765d-4adf-8112-31c55c1551ef
Save this Tunnel ID — you need it in the next step.
5
Create the config file
▼
Open the config file for editing:
sudo nano ~/.cloudflared/config.yml
Paste this inside (replace with your actual Tunnel ID and domain):
tunnel: YOUR-TUNNEL-ID
credentials-file: /root/.cloudflared/YOUR-TUNNEL-ID.json

ingress:
  - hostname: immich.yourdomain.com
    service: http://localhost:2283
  - hostname: nextcloud.yourdomain.com
    service: http://localhost:8080
  - service: http_status:404
The last line - service: http_status:404 is required — it catches any unmatched request. Don't remove it.
6
Point your subdomains to the tunnel
▼
Run once for each subdomain. This automatically creates DNS records in your Cloudflare dashboard.
cloudflared tunnel route dns home-tunnel immich.yourdomain.com
cloudflared tunnel route dns home-tunnel nextcloud.yourdomain.com
7
Run the tunnel as a background service
▼
This makes the tunnel start automatically even after a reboot.
sudo cloudflared service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared

# Check it's running fine
sudo systemctl status cloudflared
8
Test it + debug tips
▼
Open your browser and go to https://immich.yourdomain.com — it should load with HTTPS automatically.
If something's not working, check the logs:
# Watch live logs
sudo journalctl -u cloudflared -f

# Check if your service is running on the right port
curl http://localhost:2283
Browser→ DDNS → home IP→ Router port 443→ Nginx Proxy Manager→ Your service
✓ Full control over proxy ⚠ Needs port forwarding ⚠ Fails behind CGNAT ⚠ ISP may block port 443
1
Add domain to Cloudflare (same as Option A Step 1)
▼
Go to cloudflare.com → Add a Site → Free plan. Replace your registrar's nameservers with the two nameservers Cloudflare gives you. Wait 10–30 minutes.
2
Set up DDNS with ddclient
▼
ddclient watches your home IP and updates your Cloudflare DNS record automatically when it changes.
sudo apt update && sudo apt install ddclient -y
# Press Enter through the setup wizard — we'll configure manually
sudo nano /etc/ddclient.conf
Paste this into the file. Get your API token from: Cloudflare dashboard → My Profile → API Tokens → Create Token → use "Edit zone DNS" template.
protocol=cloudflare
zone=yourdomain.com
login=token
password=YOUR_CLOUDFLARE_API_TOKEN
ttl=120
yourdomain.com
sudo systemctl enable ddclient
sudo systemctl start ddclient

# Test that it's working
sudo ddclient -daemon=0 -debug -verbose -noquiet
3
Find your server's local IP
▼
You need this for port forwarding. Look for something like 192.168.1.105
ip a | grep inet
Set a DHCP reservation in your router settings so this IP never changes. Otherwise port forwarding breaks when the IP changes.
4
Set up port forwarding on your router
▼
Open 192.168.1.1 (or 192.168.0.1) in your browser → login to router admin panel → find Port Forwarding (may be under Advanced / NAT / Virtual Server).
Add these two rules, pointing to your server's local IP:
Name: HTTP  | External Port: 80  | Internal IP: 192.168.1.x | Port: 80  | Protocol: TCP
Name: HTTPS | External Port: 443 | Internal IP: 192.168.1.x | Port: 443 | Protocol: TCP
If your ISP blocks ports 80/443 or you're behind CGNAT, this won't work. Use Cloudflare Tunnel (Option A) instead — it bypasses this entirely.
5
Install Docker
▼ 
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# Log out and back in, then verify
docker --version
6
Install Nginx Proxy Manager
▼ 
mkdir ~/nginx-proxy-manager && cd ~/nginx-proxy-manager
nano docker-compose.yml
Paste this into the file:
services:
  app:
    image: jc21/nginx-proxy-manager:latest
    restart: unless-stopped
    ports:
      - 80:80
      - 443:443
      - 81:81
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
docker compose up -d
7
Configure Nginx Proxy Manager
▼
Open http://YOUR-SERVER-IP:81 in your browser.
Default login — you'll be asked to change these immediately:
Email:    admin@example.com
Password: changeme
Then go to Proxy Hosts → Add Proxy Host and fill in:
Domain Name:       immich.yourdomain.com
Scheme:            http
Forward Hostname:  localhost
Forward Port:      2283
Block Exploits:    ON (toggle)

SSL tab:
  → Request a new SSL certificate
  → Force SSL: ON
  → HTTP/2 Support: ON
  → Enter your email → Save
Repeat for each service (Nextcloud on port 8080, etc.). Let's Encrypt certificates are free and auto-renewing.
Your phone/laptop→ Tailscale VPN→ Home server→ Immich / Nextcloud
✓ No domain needed ✓ No port forwarding ✓ Most secure option ✓ Free for personal use ⚠ Only your devices can access
1
Create a Tailscale account
▼
Go to tailscale.com → sign up for free. The free plan supports up to 3 users and 100 devices — more than enough for personal use.
2
Install Tailscale on your server
▼ 
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
It prints a URL — open it in your browser and authorize the device on your Tailscale account.
3
Install Tailscale on your other devices
▼
Download the Tailscale app on your phone or laptop:
→ iOS / Android: search "Tailscale" in the App Store / Play Store
→ Windows / Mac: download from tailscale.com/download
Login with the same account. All devices are now on the same private network — no matter where in the world they are.
4
Find your server's Tailscale IP
▼ 
tailscale ip -4
You get a permanent private IP like 100.64.x.x — this never changes and is always reachable from your devices anywhere in the world.
5
Access your services
▼
From any of your devices (with Tailscale connected), just open:
http://100.64.x.x:2283    → Immich
http://100.64.x.x:8080    → Nextcloud
6
(Optional) Enable MagicDNS for nicer URLs
▼
In Tailscale admin panel (login.tailscale.com) → DNS tab → Enable MagicDNS. Your server becomes reachable by its hostname instead of IP:
http://ubuntu-server:2283   → Immich
http://ubuntu-server:8080   → Nextcloud
Browser→ yourname.duckdns.org→ Home IP (auto-updated)→ Nginx Proxy Manager→ Your service
✓ Free forever ✓ Permanent URL ⚠ URL looks less professional ⚠ Needs port forwarding
1
Get a free DuckDNS subdomain
▼
Go to duckdns.org → login with Google/GitHub → pick a name. You get yourname.duckdns.org for free forever.
Copy your token from the dashboard — you'll need it in the next step.
2
Set up auto IP updater on server
▼
This cron job runs every 5 minutes and tells DuckDNS your current home IP.
crontab -e
Add this line at the bottom (replace yourname and YOUR_TOKEN):
*/5 * * * * curl -s "https://www.duckdns.org/update?domains=yourname&token=YOUR_TOKEN&ip=" > /dev/null
Save and exit. Test it works immediately:
curl "https://www.duckdns.org/update?domains=yourname&token=YOUR_TOKEN&ip="
You should see OK in the response.
3
Port forwarding on router
▼
Open 192.168.1.1 in browser → login → find Port Forwarding → add two rules pointing to your server's local IP:
External Port 80  → Internal IP: 192.168.1.x  Port: 80  (TCP)
External Port 443 → Internal IP: 192.168.1.x  Port: 443 (TCP)
If your ISP blocks ports or you're behind CGNAT, combine DuckDNS with Cloudflare Tunnel instead of port forwarding.
4
Install Docker + Nginx Proxy Manager
▼ 
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
mkdir ~/nginx-proxy-manager && cd ~/nginx-proxy-manager
nano docker-compose.yml
Paste this into the file:
services:
  app:
    image: jc21/nginx-proxy-manager:latest
    restart: unless-stopped
    ports:
      - 80:80
      - 443:443
      - 81:81
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
docker compose up -d
5
Add proxy hosts in Nginx Proxy Manager
▼
Open http://YOUR-SERVER-IP:81 → login (admin@example.com / changeme) → Proxy Hosts → Add Proxy Host:
Domain:           immich.yourname.duckdns.org
Scheme:           http
Forward Hostname: localhost
Forward Port:     2283

SSL tab → Request new certificate → Force SSL ON
Repeat for Nextcloud using nextcloud.yourname.duckdns.org on port 8080.
Browser→ random.trycloudflare.com→ cloudflared→ Your service
✓ Zero setup ✓ Works instantly ⚠ URL changes every restart ⚠ Testing only — not permanent
1
Install cloudflared
▼ 
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb
2
Run a quick tunnel
▼
No account needed. Just point it at any local port:
cloudflared tunnel --url http://localhost:2283
Within a few seconds you'll see a random HTTPS URL like:
https://banana-coffee-random123.trycloudflare.com
Share this URL with anyone — they can access your service immediately. The tunnel stays alive as long as this command is running. Stop it with Ctrl+C.
The URL is completely random and changes every time you restart. Not suitable for permanent use — upgrade to Option A (Cloudflare Tunnel with domain) for that.
Browser→ abc123.ngrok.io→ ngrok agent→ Your service
✓ Popular for dev/testing ✓ Nice web dashboard ⚠ URL changes on free plan ⚠ 1 tunnel at a time (free)
1
Create an ngrok account
▼
Go to ngrok.com → sign up for free → copy your authtoken from the dashboard.
2
Install ngrok
▼ 
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \
  | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc > /dev/null

echo "deb https://ngrok-agent.s3.amazonaws.com buster main" \
  | sudo tee /etc/apt/sources.list.d/ngrok.list

sudo apt update && sudo apt install ngrok
3
Add your authtoken
▼ 
ngrok config add-authtoken YOUR_TOKEN_HERE
4
Start a tunnel
▼ 
ngrok http 2283
You'll get a URL like https://abc123.ngrok.io pointing to your Immich. The ngrok terminal shows live traffic stats.
5
Multiple services with a config file
▼
Create a config to run multiple tunnels at once:
nano ~/.config/ngrok/ngrok.yml
version: "2"
authtoken: YOUR_TOKEN_HERE
tunnels:
  immich:
    proto: http
    addr: 2283
  nextcloud:
    proto: http
    addr: 8080
ngrok start --all
Free plan only supports 1 tunnel at a time. You need a paid plan for running multiple tunnels simultaneously.
