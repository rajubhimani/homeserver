01

Cloudflare Tunnel

Cloudflare Tunnel lets you expose local services to the internet without port forwarding. Traffic flows from Cloudflare's edge through an outbound connection — no open inbound ports needed.

Install cloudflared

bash
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg > /dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared any main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update && sudo apt install cloudflared

Create tunnel

bash
cloudflared tunnel login
cloudflared tunnel create prajnatech

Config file

~/.cloudflared/config.yml
tunnel: <YOUR_TUNNEL_ID>
credentials-file: /home/raju/.cloudflared/<YOUR_TUNNEL_ID>.json
loglevel: error
no-autoupdate: true
grace-period: 5s

ingress:
  - hostname: "www.prajnatech.in"
    service: http://localhost:80
  - hostname: "prajnatech.in"
    service: http://localhost:80
  - hostname: "*.prajnatech.in"
    service: http://localhost:80
  - service: http_status:404

Install & start service

bash
sudo mkdir -p /etc/cloudflared
sudo cp ~/.cloudflared/config.yml /etc/cloudflared/config.yml
sudo cloudflared service install
sudo systemctl enable cloudflared
sudo systemctl start cloudflared

Set fast stop timeout

bash
sudo systemctl edit cloudflared
systemd override
[Service]
TimeoutStopSec=5
bash
sudo systemctl daemon-reload
sudo systemctl restart cloudflared
02

Docker Network & Compose Structure

Each service runs in its own compose file. A shared external Docker network lets all containers talk to each other by container name.

Create shared network

bash
docker network create homeserver
→ Run this once before starting any compose files. Every compose file references this network as external: true.

Folder structure

tree
~/docker/
├── npm/
│   └── docker-compose.yml
├── nextcloud/
│   ├── docker-compose.yml
│   └── .env
├── immich/
│   ├── docker-compose.yml
│   └── .env
└── landing/
    ├── docker-compose.yml
    └── index.html

Network block (add to every compose file)

yaml
networks:
  homeserver:
    external: true

Startup order

  • 01
    NPM first
    Nginx Proxy Manager must be up before traffic can be routed.
  • 02
    Nextcloud
    Postgres DB starts first via healthcheck dependency.
  • 03
    Immich
    Depends on Postgres and Redis — both must be healthy.
  • 04
    Landing page
    Static Nginx container, start anytime.
03

Nginx Proxy Manager

NPM is the single entry point for all HTTP traffic. Cloudflare Tunnel points to NPM on port 80, and NPM routes to each container by name.

docker-compose.yml

yaml
services:
  nginx-proxy-manager:
    image: jc21/nginx-proxy-manager:latest
    container_name: nginx-proxy-manager
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
      - "81:81"
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
    networks:
      - homeserver

networks:
  homeserver:
    external: true
→ Default login: admin@example.com / changeme — change immediately on first login.

Add proxy hosts in NPM UI (http://localhost:81)

DomainForward HostnameForward PortSSL
immich.prajnatech.inimmich-server2283none
nextcloud.prajnatech.innextcloud80none
prajnatech.inlanding80none
! No SSL needed — Cloudflare Tunnel handles TLS termination at the edge. Traffic from tunnel to NPM is plain HTTP internally.
04

Immich

Self-hosted photo and video backup with facial recognition, smart search, and mobile apps. Runs as multiple containers — server, Redis, Postgres, and optional ML.

docker-compose.yml

yaml
services:
  immich-server:
    image: ghcr.io/immich-app/immich-server:release
    container_name: immich-server
    restart: unless-stopped
    env_file: .env
    environment:
      DB_URL: ${DB_URL}
      DB_PASSWORD: ${DB_PASSWORD}
      DB_USERNAME: ${DB_USERNAME}
      DB_DATABASE_NAME: ${DB_DATABASE_NAME}
      IMMICH_SECRET: ${IMMICH_SECRET}
      REDIS_HOSTNAME: immich-redis
    volumes:
      - ${UPLOAD_LOCATION}:/usr/src/app/upload
      - /mnt/my-data/Photos:/mnt/photos:ro
      - /mnt/my-data/Videos:/mnt/videos:ro
      - /etc/localtime:/etc/localtime:ro
    depends_on:
      immich-database:
        condition: service_healthy
      immich-redis:
        condition: service_healthy
    networks:
      - homeserver

  immich-machine-learning:
    image: ghcr.io/immich-app/immich-machine-learning:release
    container_name: immich-ml
    restart: unless-stopped
    env_file: .env
    profiles:
      - ml
    volumes:
      - immich-model-cache:/cache
    networks:
      - homeserver

  immich-redis:
    image: redis:7-alpine
    container_name: immich-redis
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - homeserver

  immich-database:
    image: ghcr.io/immich-app/postgres:16-vectorchord0.5.3-pgvector0.8.1
    container_name: immich-db
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_USER: ${DB_USERNAME}
      POSTGRES_DB: ${DB_DATABASE_NAME}
    volumes:
      - ${DB_DATA_LOCATION}:/var/lib/postgresql/data
    shm_size: 128mb
    healthcheck:
      test: pg_isready --dbname="$${POSTGRES_DB}" --username="$${POSTGRES_USER}" || exit 1
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    networks:
      - homeserver

volumes:
  immich-model-cache:

networks:
  homeserver:
    external: true

Start

bash
# Without ML (recommended for low-resource setups)
docker compose up -d

# With ML (facial recognition, smart search)
docker compose --profile ml up -d

# Verify
curl http://localhost:2283/api/server/ping

Nextcloud

Self-hosted file storage and sync. Google Drive alternative with calendar, contacts, and office suite.

docker-compose.yml

yaml
services:
  nextcloud-db:
    image: postgres:16
    container_name: nextcloud-db
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - ${DATA_ROOT}/postgres-nextcloud:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - homeserver

  nextcloud:
    image: nextcloud:latest
    container_name: nextcloud
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_HOST: nextcloud-db
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      NEXTCLOUD_ADMIN_USER: ${NEXTCLOUD_ADMIN_USER}
      NEXTCLOUD_ADMIN_PASSWORD: ${NEXTCLOUD_ADMIN_PASSWORD}
      NEXTCLOUD_TRUSTED_DOMAINS: ${NEXTCLOUD_TRUSTED_DOMAINS}
      NEXTCLOUD_TRUSTED_PROXIES: nginx-proxy-manager
    volumes:
      - ${DATA_ROOT}/nextcloud:/var/www/html/data
      - ${USER_DATA_ROOT}:/mnt/seagate
    depends_on:
      nextcloud-db:
        condition: service_healthy
    networks:
      - homeserver

networks:
  homeserver:
    external: true

Trusted domains (.env)

.env
NEXTCLOUD_TRUSTED_DOMAINS=localhost 192.168.1.* 100.x.x.x *.ts.net nextcloud.prajnatech.in

Landing Page

A static landing page served by Nginx Alpine, showing all running services with links.

docker-compose.yml

yaml
services:
  landing:
    image: nginx:alpine
    container_name: landing
    restart: unless-stopped
    volumes:
      - ./index.html:/usr/share/nginx/html/index.html:ro
    networks:
      - homeserver

networks:
  homeserver:
    external: true

Deploy

bash
mkdir -p ~/docker/landing
cp index.html ~/docker/landing/
cd ~/docker/landing
docker compose up -d

Then add proxy host in NPM: prajnatech.in → landing:80

07

Cloudflare DNS

All DNS records point to the Cloudflare Tunnel. A wildcard CNAME covers all subdomains — no DNS changes needed when adding new services.

DNS records (Cloudflare Dashboard → DNS)

TypeNameTargetProxy
CNAME@<tunnel-id>.cfargotunnel.comproxied
CNAMEwww<tunnel-id>.cfargotunnel.comproxied
CNAME*<tunnel-id>.cfargotunnel.comproxied
→ Replace <tunnel-id> with your actual tunnel UUID. The wildcard * record covers all subdomains automatically — immich, nextcloud, and anything added in future.
! Do not add individual CNAME records for immich or nextcloud — NPM handles routing internally. Only the three records above are needed.

Traffic flow

flow
Browser → Cloudflare Edge (TLS) → cloudflared tunnel → NPM:80 → container
