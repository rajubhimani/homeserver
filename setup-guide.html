01

Cloudflare Tunnel

Cloudflare Tunnel lets you expose local services to the internet without port forwarding. Traffic flows from Cloudflare's edge through an outbound connection — no open inbound ports needed.

Install cloudflared

bash
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg > /dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared any main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update && sudo apt install cloudflared

Create tunnel

bash
cloudflared tunnel login
cloudflared tunnel create homeserver

Config file

~/.cloudflared/config.yml
tunnel: <YOUR_TUNNEL_ID>
credentials-file: /home/raju/.cloudflared/<YOUR_TUNNEL_ID>.json
loglevel: error
no-autoupdate: true
grace-period: 5s

ingress:
  - hostname: "www.yourdomain.com"
    service: http://localhost:80
  - hostname: "yourdomain.com"
    service: http://localhost:80
  - hostname: "*.yourdomain.com"
    service: http://localhost:80
  - service: http_status:404

Install & start service

bash
sudo mkdir -p /etc/cloudflared
sudo cp ~/.cloudflared/config.yml /etc/cloudflared/config.yml
sudo cloudflared service install
sudo systemctl enable cloudflared
sudo systemctl start cloudflared

Set fast stop timeout

bash
sudo systemctl edit cloudflared
systemd override
[Service]
TimeoutStopSec=5
bash
sudo systemctl daemon-reload
sudo systemctl restart cloudflared
02

Docker Network & Compose Structure

Each service runs in its own compose file. A shared external Docker network lets all containers talk to each other by container name.

Create shared network

bash
docker network create homeserver
→ Run this once before starting any compose files. Every compose file references this network as external: true.

Folder structure

tree
~/docker/
├── npm/
│   └── docker-compose.yml
├── nextcloud/
│   ├── docker-compose.yml
│   └── .env
├── immich/
│   ├── docker-compose.yml
│   └── .env
└── landing/
    ├── docker-compose.yml
    └── index.html

Network block (add to every compose file)

yaml
networks:
  homeserver:
    external: true

Startup tiers

  • MIN
    dozzle → cloudflared → nginx-plain → landing
    Infrastructure — must be up first. Start with sh homeserver.sh dev up min.
  • CORE
    nextcloud
    Always-on app layer. Start with sh homeserver.sh dev up core (includes min).
  • ALL
    immich, jellyfin, gitea, forgejo, gitlab, and all other services
    Optional services. Start with sh homeserver.sh dev up all. down all always stops everything.
03

Reverse Proxy

The reverse proxy is the single entry point for all HTTP traffic. Cloudflare Tunnel connects to it on port 80, and it routes requests to each container by name via the shared Docker network. Run only one proxy at a time.

Option A — nginx-plain (default)

Config-file based. Domain is injected from .env at startup via envsubst. No UI — edit the template file to add services.

bash
# Set your domain once in root .env
echo "DOMAIN=yourdomain.com" > .env

# Start nginx-plain (includes in SERVICES_MIN)
sh homeserver.sh dev up min

Each service already has a server_name <service>.${DOMAIN} block in nginx-plain/templates/default.conf.template. To add a new service, add a server block and restart.

Option B — Nginx Proxy Manager (optional)

UI-based proxy with web interface. Start instead of nginx-plain — both cannot run simultaneously.

bash
sh homeserver.sh dev up nginx

Admin UI at http://localhost:81. Default login: admin@example.com / changeme — change immediately.

04

Immich

Self-hosted photo and video backup with facial recognition, smart search, and mobile apps. Runs as multiple containers — server, Redis, Postgres, and optional ML.

docker-compose.yml

yaml
services:
  immich-server:
    image: ghcr.io/immich-app/immich-server:release
    container_name: immich-server
    restart: unless-stopped
    env_file: .env
    environment:
      DB_URL: ${DB_URL}
      DB_PASSWORD: ${DB_PASSWORD}
      DB_USERNAME: ${DB_USERNAME}
      DB_DATABASE_NAME: ${DB_DATABASE_NAME}
      IMMICH_SECRET: ${IMMICH_SECRET}
      REDIS_HOSTNAME: immich-redis
    volumes:
      - ${UPLOAD_LOCATION}:/usr/src/app/upload
      - /mnt/my-data/Photos:/mnt/photos:ro
      - /mnt/my-data/Videos:/mnt/videos:ro
      - /etc/localtime:/etc/localtime:ro
    depends_on:
      immich-database:
        condition: service_healthy
      immich-redis:
        condition: service_healthy
    networks:
      - homeserver

  immich-machine-learning:
    image: ghcr.io/immich-app/immich-machine-learning:release
    container_name: immich-ml
    restart: unless-stopped
    env_file: .env
    profiles:
      - ml
    volumes:
      - immich-model-cache:/cache
    networks:
      - homeserver

  immich-redis:
    image: redis:7-alpine
    container_name: immich-redis
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - homeserver

  immich-database:
    image: ghcr.io/immich-app/postgres:16-vectorchord0.5.3-pgvector0.8.1
    container_name: immich-db
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_USER: ${DB_USERNAME}
      POSTGRES_DB: ${DB_DATABASE_NAME}
    volumes:
      - ${DB_DATA_LOCATION}:/var/lib/postgresql/data
    shm_size: 128mb
    healthcheck:
      test: pg_isready --dbname="$${POSTGRES_DB}" --username="$${POSTGRES_USER}" || exit 1
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    networks:
      - homeserver

volumes:
  immich-model-cache:

networks:
  homeserver:
    external: true

Start

bash
# Without ML (recommended for low-resource setups)
docker compose up -d

# With ML (facial recognition, smart search)
docker compose --profile ml up -d

# Verify
curl http://localhost:2283/api/server/ping

Nextcloud

Self-hosted file storage and sync. Google Drive alternative with calendar, contacts, and office suite.

docker-compose.yml

yaml
services:
  nextcloud-db:
    image: postgres:16
    container_name: nextcloud-db
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - ${DATA_ROOT}/postgres-nextcloud:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - homeserver

  nextcloud:
    image: nextcloud:latest
    container_name: nextcloud
    restart: unless-stopped
    env_file: .env
    environment:
      POSTGRES_HOST: nextcloud-db
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      NEXTCLOUD_ADMIN_USER: ${NEXTCLOUD_ADMIN_USER}
      NEXTCLOUD_ADMIN_PASSWORD: ${NEXTCLOUD_ADMIN_PASSWORD}
      NEXTCLOUD_TRUSTED_DOMAINS: ${NEXTCLOUD_TRUSTED_DOMAINS}
      NEXTCLOUD_TRUSTED_PROXIES: nginx-proxy-manager
    volumes:
      - ${DATA_ROOT}/nextcloud:/var/www/html/data
      - ${USER_DATA_ROOT}:/mnt/seagate
    depends_on:
      nextcloud-db:
        condition: service_healthy
    networks:
      - homeserver

networks:
  homeserver:
    external: true

Trusted domains (.env)

.env
NEXTCLOUD_TRUSTED_DOMAINS=localhost 192.168.1.* 100.x.x.x *.ts.net nextcloud.yourdomain.com
06

Landing Page

A dynamic service dashboard served by Nginx Alpine. Shows all services with live status indicators — green when up, red when down. All text (domain, site name, tagline) is injected from .env at container start — no hardcoded values in the repo.

Configure landing/.env

.env
DOMAIN=yourdomain.com
SITE_NAME=MyServer
TAGLINE=Your data, your hardware, your control.
AUTHOR=Your Name
LOCATION=Your City

Start

bash
sh homeserver.sh dev up landing

How live status works

The browser fetches /health/<service> (same-origin). Nginx proxies each request to the real container on the internal Docker network. If a container is down, nginx returns 502 and the badge turns red.

flow
Browser → /health/nextcloud → nginx (landing) → nextcloud:80
→ nginx-plain routes yourdomain.com → landing:80 automatically — no manual proxy host config needed.
07

Cloudflare DNS

All DNS records point to the Cloudflare Tunnel. A wildcard CNAME covers all subdomains — no DNS changes needed when adding new services.

DNS records (Cloudflare Dashboard → DNS)

TypeNameTargetProxy
CNAME@<tunnel-id>.cfargotunnel.comproxied
CNAMEwww<tunnel-id>.cfargotunnel.comproxied
CNAME*<tunnel-id>.cfargotunnel.comproxied
→ Replace <tunnel-id> with your actual tunnel UUID. The wildcard * record covers all subdomains automatically — immich, nextcloud, and anything added in future.
! Do not add individual CNAME records for immich or nextcloud — NPM handles routing internally. Only the three records above are needed.

Traffic flow

flow
Browser → Cloudflare Edge (TLS) → cloudflared tunnel → nginx-plain:80 → container
